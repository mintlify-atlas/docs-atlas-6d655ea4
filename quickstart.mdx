<Steps>

<Step title="Install prerequisites">

Before you begin, make sure you have the following installed:

| Tool | Version | Purpose |
|------|---------|---------|
| Java | 17+ | Required to build the Maven modules locally |
| Docker | 24+ | Runs all services via Docker Compose |
| Docker Compose | 2.x | Orchestrates the five microservices |
| Maven | 3.8+ | Builds the parent POM and all modules |

<Tip>
If you only plan to run the platform (not modify source code), you only need Docker and Docker Compose — the Dockerfiles handle the Maven build inside the container.
</Tip>

Verify your Docker installation:

```bash
docker --version
docker compose version
```

</Step>

<Step title="Clone the repository">

Clone the Ubik Backend repository and navigate to the Docker Compose root:

```bash
git clone https://github.com/your-org/ubik-backend.git
cd ubik-backend/microservicios/microreactivo
```

<Note>
All `docker compose` commands in this guide must be run from the `microservicios/microreactivo/` directory, which is where `docker-compose.yml` lives.
</Note>

</Step>

<Step title="Configure environment variables">

Create a `.env` file in `microservicios/microreactivo/` with the following variables. Docker Compose reads this file automatically at startup.

```bash
# .env

# ── Database (Azure Flexible Server or any MySQL-compatible host) ────────────
SPRING_R2DBC_URL=r2dbc:mysql://your-db-host:3306/ubik_db
SPRING_R2DBC_USERNAME=ubik_user
SPRING_R2DBC_PASSWORD=your_db_password

# ── JWT ─────────────────────────────────────────────────────────────────────
JWT_SECRET=your_very_long_random_secret_key_at_least_32_chars
JWT_EXPIRATION=86400000   # 24 hours in milliseconds (default)

# ── Cloudinary (image uploads for motel/room photos) ────────────────────────
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret

# ── Role IDs (must match the IDs stored in your roles table) ────────────────
ROLE_ID_ADMIN=1
ROLE_ID_PROPERTY_OWNER=2
ROLE_ID_USER=3

# ── Email (SMTP credentials for the Notification Service) ───────────────────
MAIL_USERNAME=your_email@example.com
MAIL_PASSWORD=your_smtp_password

# ── MercadoPago (Payment Service) ───────────────────────────────────────────
MERCADOPAGO_ACCESS_TOKEN=your_mp_access_token
MERCADOPAGO_WEBHOOK_SECRET=your_mp_webhook_secret
MERCADOPAGO_CLIENT_ID=your_mp_client_id
MERCADOPAGO_CLIENT_SECRET=your_mp_client_secret
MERCADOPAGO_REDIRECT_URI=https://your-domain.com/oauth/callback
MP_FEE_PERCENT=10   # Platform fee percentage (default 10)
```

<Warning>
Never commit your `.env` file to source control. It contains secrets. Add it to `.gitignore` immediately.
</Warning>

</Step>

<Step title="Build and run with Docker Compose">

Build all service images and start the stack:

```bash
docker compose up --build
```

To run in detached mode (background):

```bash
docker compose up --build -d
```

Docker Compose starts services in dependency order:
1. `usermanagement-service`, `motel-management-service`, and `notification-service` start first.
2. `payment-service` waits for both `motel-management-service` and `notification-service` to be healthy.
3. `gateway` waits for all three foundational services to be healthy before it starts.

<Note>
The initial build downloads base images and compiles all Maven modules. Expect the first run to take 3–5 minutes. Subsequent starts are much faster.
</Note>

</Step>

<Step title="Verify services are healthy">

Each service exposes a Spring Boot Actuator health endpoint. Poll them to confirm everything is running:

```bash
# API Gateway (the single entry point — check this first)
curl http://localhost:8080/actuator/health

# Individual services (accessible directly for debugging)
curl http://localhost:8081/actuator/health   # User Management
curl http://localhost:8083/actuator/health   # Motel Management
curl http://localhost:8084/actuator/health   # Notification Service
curl http://localhost:8085/actuator/health   # Payment Service
```

A healthy service responds with:

```json
{"status":"UP"}
```

You can also check container status:

```bash
docker compose ps
```

All five containers should show `healthy` in the `STATUS` column before you proceed.

<Warning>
The `gateway` container only starts after `usermanagement-service`, `motel-management-service`, and `notification-service` report healthy. If the gateway is still starting, wait up to 90 seconds for its `start_period` to elapse.
</Warning>

</Step>

<Step title="Make your first API call">

All API traffic goes through the gateway on port **8080**. Start by registering a user, then log in to obtain a JWT.

**Register a new user**

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "alice",
    "password": "secret123",
    "email": "alice@example.com",
    "phoneNumber": "+57 300 000 0000",
    "anonymous": false,
    "roleId": 3,
    "longitude": -74.0721,
    "latitude": 4.7110,
    "birthDate": "1995-06-15"
  }'
```

A successful registration returns `HTTP 201 Created`.

**Log in**

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "alice",
    "password": "secret123"
  }'
```

The response body is a signed JWT string:

```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**Use the token in authenticated requests**

Pass the JWT in the `Authorization` header for any protected endpoint:

```bash
TOKEN="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

curl http://localhost:8080/api/auth/motels/my-motels \
  -H "Authorization: Bearer $TOKEN"
```

<Tip>
Public GET endpoints (`/api/motels/**`, `/api/rooms/**`, `/api/services/**`) do not require a token. Only write operations and authenticated routes require the `Authorization` header.
</Tip>

</Step>
