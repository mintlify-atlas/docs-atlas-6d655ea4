---
title: "Environment variables"
description: "Complete reference for every environment variable consumed by the Ubik Backend Docker Compose stack."
---

Docker Compose reads variables from a `.env` file located in `microservicios/microreactivo/`. Create this file before running `docker compose up`.

<Warning>
Never commit your `.env` file to source control. It contains database credentials, JWT signing keys, and payment processor secrets. Add `.env` to `.gitignore` immediately after creating it.
</Warning>

## Gateway

The gateway runs on port **8080** and acts as the single entry point for all API traffic. It validates JWTs on every request and enforces role-based access using role ID values.

| Variable | Description | Example | Required |
|----------|-------------|---------|----------|
| `SPRING_PROFILES_ACTIVE` | Spring profile. Set to `docker` when running in Docker Compose. | `docker` | Yes |
| `JWT_SECRET` | Secret key used to sign and verify JWTs. Must match the value used in User Management. Use a cryptographically random string of at least 32 characters. | `a8f3d...` | Yes |
| `JWT_EXPIRATION` | JWT validity duration in milliseconds. Defaults to `86400000` (24 hours). | `86400000` | No |
| `ROLE_ID_ADMIN` | Numeric ID of the admin role in the `roles` table. Used to authorize admin endpoints. | `1` | Yes |
| `ROLE_ID_PROPERTY_OWNER` | Numeric ID of the property owner role. | `2` | Yes |
| `ROLE_ID_USER` | Numeric ID of the regular user role. | `3` | Yes |
| `JAVA_OPTS` | JVM memory flags passed to the container entrypoint. | `-Xmx512m -Xms256m` | No |

<Warning>
`JWT_SECRET` must be identical in both the gateway and User Management service. A mismatch causes all authenticated requests to fail with 401.
</Warning>

## User Management

The User Management service runs on port **8081**. It handles registration, authentication, and user profile management, and issues signed JWTs.

| Variable | Description | Example | Required |
|----------|-------------|---------|----------|
| `SPRING_PROFILES_ACTIVE` | Spring profile. | `docker` | Yes |
| `SPRING_R2DBC_URL` | R2DBC connection URL for the MySQL database. | `r2dbc:mysql://your-host:3306/ubik_db` | Yes |
| `SPRING_R2DBC_USERNAME` | Database username. | `ubik_user` | Yes |
| `SPRING_R2DBC_PASSWORD` | Database password. | `s3cr3t` | Yes |
| `JWT_SECRET` | Secret key for signing JWTs. Must match the gateway's value. | `a8f3d...` | Yes |
| `JWT_EXPIRATION` | JWT validity in milliseconds. Defaults to `86400000`. | `86400000` | No |
| `JAVA_OPTS` | JVM memory flags. | `-Xmx512m -Xms256m` | No |

## Motel Management

The Motel Management service runs on port **8083**. It manages motel listings, rooms, service offerings, reservations, and photo uploads via Cloudinary.

| Variable | Description | Example | Required |
|----------|-------------|---------|----------|
| `SPRING_PROFILES_ACTIVE` | Spring profile. | `docker` | Yes |
| `SPRING_R2DBC_URL` | R2DBC connection URL for the MySQL database. | `r2dbc:mysql://your-host:3306/ubik_db` | Yes |
| `SPRING_R2DBC_USERNAME` | Database username. | `ubik_user` | Yes |
| `SPRING_R2DBC_PASSWORD` | Database password. | `s3cr3t` | Yes |
| `CLOUDINARY_CLOUD_NAME` | Your Cloudinary account's cloud name, used for motel and room photo storage. | `my-cloud` | Yes |
| `CLOUDINARY_API_KEY` | Cloudinary API key. | `123456789012345` | Yes |
| `CLOUDINARY_API_SECRET` | Cloudinary API secret. | `abcDEF_xyz...` | Yes |
| `ROLE_ID_ADMIN` | Numeric ID of the admin role. Must match the gateway's value. | `1` | Yes |
| `ROLE_ID_PROPERTY_OWNER` | Numeric ID of the property owner role. Must match the gateway's value. | `2` | Yes |
| `ROLE_ID_USER` | Numeric ID of the regular user role. Must match the gateway's value. | `3` | Yes |
| `JAVA_OPTS` | JVM memory flags. | `-Xmx512m -Xms256m` | No |

<Note>
The role ID variables (`ROLE_ID_ADMIN`, `ROLE_ID_PROPERTY_OWNER`, `ROLE_ID_USER`) must be consistent across the gateway and Motel Management service. These values must match the IDs stored in your database's `roles` table.
</Note>

## Notification Service

The Notification Service runs on port **8084**. It sends transactional emails (booking confirmations, password resets) using Gmail SMTP with STARTTLS.

| Variable | Description | Example | Required |
|----------|-------------|---------|----------|
| `SPRING_PROFILES_ACTIVE` | Spring profile. | `docker` | Yes |
| `MAIL_USERNAME` | Gmail address used as the SMTP sender. | `notifications@example.com` | Yes |
| `MAIL_PASSWORD` | Gmail App Password (not your Google account password). Generate one in your Google Account security settings under "App passwords". | `abcd efgh ijkl mnop` | Yes |
| `JAVA_OPTS` | JVM memory flags. | `-Xmx512m -Xms256m` | No |

<Warning>
Use a [Gmail App Password](https://myaccount.google.com/apppasswords), not your regular Google account password. Regular passwords are rejected when two-factor authentication is enabled on the account.
</Warning>

<Note>
The Notification Service deliberately disables Spring Boot's mail health indicator (`management.health.mail.enabled: false`). This prevents the Gmail SMTP connection from being used as a liveness signal, which would cause health check failures when Gmail throttles connections.
</Note>

## Payment Service

The Payment Service runs on port **8085**. It integrates with MercadoPago as a marketplace platform, handling payment preference creation, OAuth-based seller onboarding, and webhook processing.

| Variable | Description | Example | Required |
|----------|-------------|---------|----------|
| `SPRING_PROFILES_ACTIVE` | Spring profile. | `docker` | Yes |
| `SPRING_R2DBC_URL` | R2DBC connection URL for the MySQL database. | `r2dbc:mysql://your-host:3306/ubik_db` | Yes |
| `SPRING_R2DBC_USERNAME` | Database username. | `ubik_user` | Yes |
| `SPRING_R2DBC_PASSWORD` | Database password. | `s3cr3t` | Yes |
| `MERCADOPAGO_ACCESS_TOKEN` | Marketplace platform access token from MercadoPago. | `APP_USR-...` | Yes |
| `MERCADOPAGO_WEBHOOK_SECRET` | Secret used to verify the HMAC signature on incoming MercadoPago webhook notifications. | `whsec_...` | Yes |
| `MERCADOPAGO_CLIENT_ID` | Your MercadoPago application ID. | `1234567890123456` | Yes |
| `MERCADOPAGO_CLIENT_SECRET` | Your MercadoPago application secret. | `aB1cD2eF3gH4...` | Yes |
| `MERCADOPAGO_REDIRECT_URI` | OAuth callback URL registered in your MercadoPago application. Must be a publicly reachable HTTPS URL. | `https://your-domain.com/api/payments/oauth/callback` | Yes |
| `MP_FEE_PERCENT` | Platform fee percentage deducted from each transaction. Defaults to `10`. | `10` | No |
| `JAVA_OPTS` | JVM memory flags. | `-Xmx512m -Xms256m` | No |

<Warning>
`MERCADOPAGO_ACCESS_TOKEN`, `MERCADOPAGO_CLIENT_SECRET`, and `MERCADOPAGO_WEBHOOK_SECRET` are highly sensitive. Exposure of these values allows third parties to initiate charges or intercept payment events on your behalf.
</Warning>

## Shared variables

Several variables appear in multiple services and must have consistent values across the entire stack.

| Variable | Used by | Notes |
|----------|---------|-------|
| `JWT_SECRET` | Gateway, User Management | Must be identical in both services |
| `JWT_EXPIRATION` | Gateway, User Management | Should be consistent; defaults to `86400000` |
| `SPRING_R2DBC_URL` | User Management, Motel Management, Payment Service | All three services connect to the same database host |
| `SPRING_R2DBC_USERNAME` | User Management, Motel Management, Payment Service | Database username |
| `SPRING_R2DBC_PASSWORD` | User Management, Motel Management, Payment Service | Database password |
| `ROLE_ID_ADMIN` | Gateway, Motel Management | Must match the `roles` table |
| `ROLE_ID_PROPERTY_OWNER` | Gateway, Motel Management | Must match the `roles` table |
| `ROLE_ID_USER` | Gateway, Motel Management | Must match the `roles` table |

## Example `.env` file

```bash
# ── Database (Azure MySQL Flexible Server) ────────────────────────────────────
SPRING_R2DBC_URL=r2dbc:mysql://your-server.mysql.database.azure.com:3306/ubik_db
SPRING_R2DBC_USERNAME=ubik_user
SPRING_R2DBC_PASSWORD=your_db_password

# ── JWT ───────────────────────────────────────────────────────────────────────
JWT_SECRET=your_very_long_random_secret_key_at_least_32_chars
JWT_EXPIRATION=86400000

# ── Role IDs (must match your roles table) ────────────────────────────────────
ROLE_ID_ADMIN=1
ROLE_ID_PROPERTY_OWNER=2
ROLE_ID_USER=3

# ── Cloudinary ────────────────────────────────────────────────────────────────
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret

# ── Email (Gmail SMTP) ────────────────────────────────────────────────────────
MAIL_USERNAME=notifications@your-domain.com
MAIL_PASSWORD=your_gmail_app_password

# ── MercadoPago ───────────────────────────────────────────────────────────────
MERCADOPAGO_ACCESS_TOKEN=APP_USR-your-access-token
MERCADOPAGO_WEBHOOK_SECRET=your_webhook_secret
MERCADOPAGO_CLIENT_ID=your_client_id
MERCADOPAGO_CLIENT_SECRET=your_client_secret
MERCADOPAGO_REDIRECT_URI=https://your-domain.com/api/payments/oauth/callback
MP_FEE_PERCENT=10
```
